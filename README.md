# otel-receiver-nvidia

An [OpenTelemetry Collector](https://opentelemetry.io/docs/collector/) **metrics receiver** for NVIDIA GPUs on Linux hosts. It runs the `nvidia-smi` command line tool on a configurable interval and turns the result into OpenTelemetry metrics, one set per GPU.

This repository is meant to be compiled into your own Collector distribution with the [OpenTelemetry Collector Builder (OCB)](https://opentelemetry.io/docs/collector/custom-collector/) — see [Building a custom collector](#building-a-custom-collector-with-ocb) below.

## Metrics

| Metric                          | Unit | Type         | Description                                        |
| ------------------------------- | ---- | ------------ | -------------------------------------------------- |
| `nvidia.gpu.utilization`        | %    | Gauge Double | GPU compute utilization (`utilization.gpu`).       |
| `nvidia.gpu.memory.utilization` | %    | Gauge Double | GPU memory used as a percent of total (`used/total`).|
| `nvidia.gpu.memory.total`       | By   | Gauge Int    | Total installed GPU memory.                        |
| `nvidia.gpu.memory.used`        | By   | Gauge Int    | GPU memory in use.                                 |
| `nvidia.gpu.memory.free`        | By   | Gauge Int    | GPU memory free (unused).                          |

Each metric carries the resource attributes `nvidia.gpu.index`, `nvidia.gpu.uuid`, and `nvidia.gpu.name`.

Full receiver documentation: [`nvidiareceiver/README.md`](./nvidiareceiver/README.md) and [`nvidiareceiver/documentation.md`](./nvidiareceiver/documentation.md).

## Requirements

- An NVIDIA GPU with the NVIDIA driver installed.
- `nvidia-smi` available on the Collector host's `PATH` (or set `binary_path`).
- Linux (the component is not supported on macOS).

## Building a custom collector with OCB

[OCB](https://opentelemetry.io/docs/collector/custom-collector/) builds a Collector binary from a manifest that lists the components to include. Add this receiver to the `receivers` section of your manifest. Because the package lives in the `nvidiareceiver/` subdirectory of the module, set both `gomod` (the module) and `import` (the package).

A ready-to-use manifest is provided in [`examples/builder-config.yaml`](./examples/builder-config.yaml):

```yaml
receivers:
  - gomod: github.com/pierre-prevoteau/otel-receiver-nvidia v0.1.0
    import: github.com/pierre-prevoteau/otel-receiver-nvidia/nvidiareceiver
```

### 1. Install OCB

```bash
go install go.opentelemetry.io/collector/cmd/builder@v0.154.0
```

### 2. Build

```bash
builder --config examples/builder-config.yaml
```

This produces the binary under `./otelcol-nvidia/` (per `dist.output_path` in the manifest).

### Using a published version vs. local source

- **Published version** — once you tag a release (`git tag v0.1.0 && git push --tags`), reference it directly with `gomod: github.com/pierre-prevoteau/otel-receiver-nvidia v0.1.0`.
- **Local source (before publishing)** — keep the `replaces` directive in the manifest so OCB compiles your working tree instead of fetching a tag. The path is relative to the generated collector's `go.mod` (the `output_path` directory):

```yaml
replaces:
  - github.com/pierre-prevoteau/otel-receiver-nvidia => ../
```

## Running the collector

A sample runtime config is in [`examples/otelcol.yaml`](./examples/otelcol.yaml):

```bash
./otelcol-nvidia/otelcol-nvidia --config examples/otelcol.yaml
```

```yaml
receivers:
  nvidia:
    collection_interval: 30s

exporters:
  debug:
    verbosity: detailed

service:
  pipelines:
    metrics:
      receivers: [nvidia]
      exporters: [debug]
```

## Development

This project uses [`mdatagen`](https://github.com/open-telemetry/opentelemetry-collector/tree/main/cmd/mdatagen) to generate the `internal/metadata` package, `documentation.md`, and the generated tests from `nvidiareceiver/metadata.yaml`.

```bash
# Regenerate code from metadata.yaml
make generate

# Run tests (and re-create the golden file after changing emitted metrics)
make test
WRITE_GOLDEN=true go test ./...

# Tidy modules
make tidy
```

> Note: `mdatagen` cannot be installed with `go install ...@version` because its module ships with `replace` directives. The `make generate` target works around this by running it from a throwaway tool module (Go 1.24+ `go tool`).
